A semântica na Web e os dados estruturados no Google: Apresentarei os conceitos principais de Web Semântica e os primeiros passos para começar a publicar dados com semântica na Web, mostrarei também como o Google tem utilizado dados estruturados em seus produtos e serviços.

Jean Carlo Emer

Como encarar o desenvolvimento front-end: A todo tempo surgem novas formas de pensar e estruturar interfaces para a Internet. Esta palestra toma um olhar crítico a respeito de ferramentas, técnicas e código front-end em geral. Nosso objetivo será aprender a direcionar esforços para alcançar melhores resultados.

Igor Ribeiro

Uma visão geral sobre banco de dados em tempo real: Essa palestra irá cobrir uma alternativa de como desenvolver um código Front-end sem ter que se preocupar com o desenvolvimento de um servidor. O uso de um serviço DBaaS (Banco de Dados como serviço) nos permite focar somente no que os desenvolvedores JavaScript gostam - codificar só em JS. Não importa qual framework você já está familiarizado: ReactJS, AngularJS, Backbone ou EmberJS.

Zeno Rocha

Web APIs You Didn’t Know Existed: You’ve probably heard about the amazing things that HTML5 can do. Maybe you already saved some data using Local Storage, fetched user coordinates using Geolocation, or even drew graphs and other objects using Canvas. But did you know there are many more new JavaScript APIs in the Web Platform? In this talk, we’ll see how to leverage the power of those APIs in order to create better experiences for your users.

17:00 pm

Maximiliano Firtman

Extreme Web Performance In this talk we will understand why performance is the key issue on a web, the most important differences on mobile that we might be underestimating and how to achieve extreme web performance. We'll review new metrics such as Speed Index, the RAIL goals that Google promotes and how you can harm your performance and conversions if you don't pay enough attention to the details. We will see what we can do from the front-end side to help Web Performance Optimization, including the updated checklist of techniques to improve experience and perception, including: the usage of HTTP/2, HTTP Client Hints, how to replace PNGs and JPEGs, better compression encoding algorithms available today, browser's prefetching and lazy loading new abilities and how Service Workers can help improving conversion and reduce user's frustration. We'll cover some extreme new ideas, such as serving ATF content in 14Kb, using magician tricks or how to use new APIs to make decisions on the fly.
